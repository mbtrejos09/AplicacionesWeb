Flexbox vs Grid

Tipo de Dato Flexbox(display:flex) Grid (display:grid)
Caracteristicas
  • Una dimensión
  • Centrado en el contenido
  • Aliación en filas y columnas
  • Mayor simplicidad.
  • Configurable como Fila, Fila inversa, Columna,Columna inversa
  • Bidimensional
  • Diseño más estructurado y preciso
  • Permite definir tamaños de celdas específicas
  • Alinea elementos en ambas direcciones con grid-template-areas, grid-template-columns y grid-template-rows.
  • Ideal para layouts de página completos
Principales Diferencias
  • Creado para diseños de una dimensión, en una fila o una columna
  • Diseño más simple y fácil de usar
  • Pensado para el diseño bidimensional, en varias filas y columnas al mismo tiempo
Similitudes
  • Establecer primero el diseño del elemento padre : al diseñar un diseño flexible o de cuadrícula, el elemento padre es el que tiene el estilo exacto en cuestión.
  • Ambos son diseños adaptables : independientemente del diseño que decida utilizar para su aplicación, siempre que diseñe las propiedades correctamente, ambos tipos de diseño responden a diferentes tamaños de pantalla.
Casos de Uso
  • Barras de navegación
  • Centrado horizontal o vertical
  • Espaciar los elementos de manera uniforme dentro de un contenedor
  • Contenido dinámico que puede tener que ajustarse o tener que ajustarse en tamaño
  • Útil para menús de navegación, listas, tarjetas y botones alineados.
  • Diseños de cuadrícula complejos o los elementos deben ir en filas y columnas.
  • Diseños más adaptables, es decir el diseño da forma al contenido, se adopta un enfoque que prioriza el diseño.
  • Alineación precisa de elementos.
  • Cuando desee diseñar el diseño combinado de la página.
  • Para diseños de página completos.
Ejemplos
  • div {class="flex-container"} div{1} div{2} div{3} .flex-container { display: flex; }
  • .grid-container { display: grid; grid-template-areas: "header header" "sidebar main" "footer footer"; grid-template-columns: 200px 1fr; grid-template-rows: 60px 1fr 50px; gap: 10px; height: 100vh; } .header { grid-area: header; background: #333; color: white; text-align: center; padding: 15px; } .sidebar { grid-area: sidebar; background: #f4f4f4; padding: 15px; } .main-content { grid-area: main; background: #ddd; padding: 15px; } .footer { grid-area: footer; background: #333; }

Mejores prácticas de nombramiento de CSS

Nombre Explicación Ventajas Desventajas Ejemplos
lowerCamelCase
  • Se usa para definir clases con múltiples palabras sin espacios, donde la primera letra es minúscula y cada palabra siguiente empieza con mayúscula.
  • Comúnmente usado en JavaScript y frameworks como React.
  • Evita el uso de caracteres especiales, reduciendo errores en ciertas plataformas.
  • Compatible con classList en JavaScript sin transformaciones adicionales.
  • Menos legible en comparación con kebab-case.
  • No es el estándar en HTML/CSS, lo que puede generar inconsistencias en equipos grandes.
  • .nombreClaseEjemplo {}
kebab-case
  • Se usa para definir clases en CSS separando las palabras con guiones.
  • Más legible y fácil de entender en HTML y CSS.
  • Facilita la búsqueda y mantenimiento del código.
  • Compatible con todos los navegadores y herramientas de CSS.
  • Es el estándar en HTML y CSS, lo que mejora la compatibilidad.
  • No es directamente compatible con classList en JavaScript sin transformaciones.
  • Puede volverse difícil de escribir si hay muchas palabras en la clase.
  • .mi-clase-ejemplo {}
BEM (Block-Element-Modifier)
  • Se basa en bloques, elementos y modificadores para mejorar la legibilidad del CSS.
  • Diseño adaptable.
  • Organiza el código de forma clara.
  • Facilita la reutilización de componentes.
  • Puede generar nombres de clases largos y difíciles de manejar.
  • Requiere disciplina en su aplicación.
  • .bloque__elemento--modificador {}
  • .boton__icono--grande {}
OOCSS (Object-oriented CSS)
  • Divide los estilos en estructuras reutilizables para mejorar la modularidad.
  • Fomenta la reutilización de estilos, reduciendo la duplicación de código.
  • Mejora la separación entre estructura y apariencia.
  • Facilita el mantenimiento al dividir estilos en componentes reutilizables.
  • Reduce la carga de CSS al aprovechar componentes comunes.
  • Puede ser difícil de entender al principio.
  • No sigue una estructura estricta, lo que puede generar inconsistencias en proyectos grandes.
  • .card-title {}
  • .card-content {}
ITCSS (Inverted Triangle CSS)
  • Organiza los estilos en capas, desde los más generales hasta los más específicos.
  • Mejora la organización del código en proyectos grandes.
  • Reduce la sobrecarga de estilos innecesarios.
  • Alineación precisa de elementos.
  • Evita problemas de especificidad en CSS.
  • Facilita la escalabilidad al mantener una estructura clara.
  • Puede ser complejo de implementar en proyectos pequeños.
  • Requiere una buena planificación previa.
  • .layout-header {}
  • .layout-footer {}
Utility-First CSS (Ej. Tailwind CSS)
  • Usa clases utilitarias predefinidas para diseño rápido sin escribir CSS personalizado.
  • Permite desarrollar más rápido sin necesidad de escribir CSS personalizado.
  • Facilita la creación de interfaces consistentes.
  • Alineación precisa de elementos.
  • Elimina la necesidad de escribir archivos CSS grandes.
  • Evita la sobrecarga de estilos al usar solo lo necesario.
  • No es tan flexible para personalizaciones avanzadas sin configuración adicional.
  • Dependencia del framework para realizar cambios.
  • Puede generar código HTML muy largo y difícil de leer.
  • class="bg-blue-500 text-white px-4 py-2 rounded"
SMACSS (Scalable and Modular Architecture for CSS)
  • Clasifica los estilos en categorías como Base, Layout, Module, State y Theme.
  • Facilita la escalabilidad de proyectos grandes.
  • Organización clara del código en diferentes categorías.
  • Reduce la probabilidad de conflictos entre estilos.
  • Cuando desee diseñar el diseño combinado de la página.
  • Fomenta el uso de estilos modulares y reutilizables.
  • Requiere tiempo para aprender y aplicar correctamente.
  • Puede agregar complejidad a proyectos pequeños.
  • .module-button {}
  • .state-hidden {}
